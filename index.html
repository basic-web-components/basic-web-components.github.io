Basic Web Components

The Basic Web Components project seeks to provide a comprehensive set of solid, well-designed web components that implement very common user interface patterns. Each component can be used as is, or as the foundation for new components.

See the Basic Web Components repository on GitHub for complete documentation.

basic-arrow-selection

Adds prominent left and right arrow buttons to a wrapped child such as a basic-carousel. By default, the arrow buttons are shown on devices with a mouse-like pointing device.

basic-animation-stage

Provides animated transitions for selection changes. An animation can be provided specified with keyframes, or by naming a stock effect. The same animation can be shown at an arbitrary point, generally used to reflect a user-controlled touch or trackpad drag operation in progress.

This component is a programmatic rendering surface, and comes with no interactivity of its own. This demo adds an instance of basic-page-dots. For a fully interactive carousel that includes touch support, see basic-carousel.

basic-autosize-textarea

A text area that grows in size to contain its contents, unlike a standard <textarea> element that always remains the same height.

basic-carousel

An implementation of the carousel user interface pattern, commonly used for navigating laterally between images, pages, and other elements. This pattern presents the user with a linear sequence of elements, through which the user can navigate with a variety of input methods: touch, trackpad, and keyboard.

The basic-carousel family of components includes a number of auxiliary components for adding additional means of navigation. See basic-arrow-selection, basic-page-dots, basic-slideshow-timer, and basic-tab-strip.

basic-collapsible-panel

A panel which can be expanded/collapsed with simple CSS height animation. It handles only the duties of collapsing and expanding; you must provide a user interface to trigger that change yourself. (This demo includes a button for that purpose.)

basic-current-anchor

An anchor (link) that highlights itself when its destination matches the current location. Such links often come up in toolbars, side bars, and other navigation elements in which you want the user to be able to quickly confirm their location within your app.

basic-list-box

A single-selection list box with good keyboard support. The Up/Down, Page Up/Down, and Home/End keys work as expected. You can also type the beginning of an item to select it. This list box also has good accessibility support.

basic-modes

Shows exactly one child element at a time. This can be useful in cases where an element has multiple modes that present substantially different user interfaces. This component provides no interactivity of its own. In this demo, the component has been wrapped with an instance of basic-arrow-selection.

basic-page-dots

Augments a wrapped element such as a basic-carousel or basic-slideshow with a set of small dots, one for each element in the list. The user can use these dots to see how many elements are in the list, and to identify their current position in the list.

basic-play-controls

Augments a wrapped slideshow, audio playlist, etc., with controls for play/pause, and to select the previous/next item.

basic-slideshow

A slideshow with animated transitions. This can be used on its own, or combined with a component like basic-play-controls to give your user control over playback.

basic-slideshow-timer

Adds slideshow semantics to a wrapped element (here, a basic-carousel with basic-page-dots), allowing the selection to advance on a timed basis. If the user directly manipulates the selection, the slideshow timer starts over, giving the user time to view the selected item.

basic-sliding-viewport

Presents list items in a viewport such that only a single item is visible at a time, and shows changes in seleciton with a simple sliding effect. This is somewhere between basic-modes (which provides no visual transition) and basic-animation-stage (a more complex rendering surface that requires the Web Animation API). This component provides no interactivity of its own; in this demo, it's been wrapped with an instance of basic-arrow-selection.

basic-spread-items

Spreads out a set of items horizontally so they take equal space. This is a relatively low-level layout component used by basic-sliding-viewport.

basic-tab-strip

Wraps a list component with a strip of tabs, one for each item in the list. Selecting a tab will select the corresponding item. Typically combined with a component like basic-animation-stage or basic-modes. The latter combination is so common that it is made available as the single component basic-tabs.

basic-tabs

A set of pages with a tab strip governing which page is shown. This stock combination puts together an instance of basic-tab-strip with an instance of basic-modes. To create more complex arrangements, you can use either of those elements on its own.
